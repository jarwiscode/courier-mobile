MVP отслеживания курьеров (законно, на основе согласия)

Обзор
- Этот проект предоставляет законную систему отслеживания курьеров в реальном времени, основанную на их явном согласии.
- Он собирает обновления геолокации из приложения на устройстве в активном режиме (foreground) и отображает их на дашборде.
- Система НЕ перехватывает телефонные звонки и не записывает аудио телефонной сети (PSTN). Если нужны звонки/запись, используйте VoIP внутри приложения с явным согласием.

Компоненты
- server: сервер Node.js с REST‑эндпоинтом для приёма обновлений геолокации и Socket.IO для трансляции на дашборды.
- dashboard: простая веб‑страница, обслуживаемая сервером, для визуализации обновлений в реальном времени.
- mobile: руководство по интеграции для отправки периодических обновлений геолокации с устройств курьеров.

Быстрый старт
1) Установите зависимости сервера:
   - cd server
   - npm install
   - npm start
2) Откройте дашборд:
   - http://localhost:8080/dashboard
3) Отправьте тестовую локацию (пример):
   - curl -X POST http://localhost:8080/v1/locations \\
     -H 'Content-Type: application/json' \\
     -d '{ "courierId": "c1", "lat": 40.7128, "lng": -74.0060, "speed": 8.3, "heading": 135, "jobId": "job-123", "battery": 0.76, "timestamp": 1730899200000 }'

Правовые аспекты, согласие и требования платформ
- Получайте явное и информированное согласие курьеров; прозрачно сообщайте о непрерывном сборе геолокации и целях обработки.
- Соблюдайте правила ОС: на Android используйте видимый foreground‑сервис и запрашивайте разрешения на геолокацию в активном режиме. На iOS используйте режимы .whenInUse или .always с корректными пояснениями в Info.plist и видимым системным индикатором.
- Предоставьте в приложении переключатель, позволяющий приостанавливать/останавливать отправку вне рабочих часов.
- Храните данные только столько, сколько необходимо; используйте контроль доступа и шифрование при передаче (HTTPS/WSS) и при хранении в продакшене.
- Не записывайте и не получайте доступ к аудио телефонной сети (PSTN); ОС запрещают скрытый перехват звонков.

Следующие шаги
- Замените хранилище в памяти на базу данных (например, Postgres) и добавьте аутентификацию.
- Используйте HTTPS и защищённые WebSocket‑соединения (WSS) в продакшене.
- Добавьте геозоны и оповещения SLA (опоздания, отклонение от маршрута).
- Постройте полноценный админ‑дашборд (слои карты, фильтры, воспроизведение истории).
